## Overview
Featuring Python, Go, Rust, C++, and more. This project focuses on physics simulation and input validation across different runtime environments.

## Why Multi-language? (The "Algorithmic Hack")
This project is more than just a physics simulation. By implementing the same AP11 logic across multiple languages, I am testing two specific hypotheses:

### 1. Polyglot Verification
Comparing how different stacks—**Python, Go, Rust, and C++**—handle the same core algorithm. It’s an exercise in identifying the "essence of logic" by stripping away language-specific syntax and focusing on runtime characteristics.

### 2. Platform Optimization (The Real Reason)
Let’s be honest: modern recruitment platforms and crawlers prioritize **"Language Diversity"** and **"Repository Structure"** in their scoring algorithms. This multi-language expansion is a deliberate tactical move to optimize for those metrics.

**It's a "meta-hack" for a recruitment system that prefers stats over substance.**
